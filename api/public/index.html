API/AUTH

api/auth/register

REQUEST
{
"username": "studentnumberone",
"password": "thisispassword",
"name": "Student One",
"email": "student1@awesomeschool.com",
"roleId": 1
}
RESPONSE
{
"id": 4,
"name": "Student One",
"username": "studentnumberone",
"roleId": 1
}

api/auth/login

REQUEST
{
"username": "studentnumberone",
"password": "thisispassword",
"roleId": 1
}
RESPONSE
{
"token": "$#Imagine%this@!is*&a^$**complicated#@TOKEN#@",
"message": "Welcome Student Two",
"username": "student",
"role": "Student"
}
  • Token received from login is hashed with ID and roles, which restricted particular users from doing certain tasks
  • e.g.: students CANNOT post solutions, or staff cannot post tickets
  • When making POST or PUT request, user's ID will be decoded from this TOKEN so there's no need to make a request with user id

POST api/tickets
  • Create a new ticket
  • Can only be created by STUDENTS
  • ASKER will be decoded from Token so this action is restricted only to students
  • CREATEAT is autogenerated
  • Any attempts to crete by STAFF will be blocked and result in PERMISSION DENIED
  • RETURN new ticket's id and title
